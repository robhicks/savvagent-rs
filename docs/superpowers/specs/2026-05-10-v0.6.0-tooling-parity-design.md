# v0.6.0 — Tooling Parity — Design

**Issue:** [#10](https://github.com/robhicks/savvagent-rs/issues/10)
**Status:** approved by user 2026-05-10
**Target version:** 0.6.0

## Theme

Tooling parity. Closes the §10 backlog "Tools" bullet (`tool-grep`, structured
`tool-edit`, richer `glob`) carried since v0.4, plus the SPP `list_models` work
needed to make `/model` validation real instead of optimistic. Mirrors the
v0.4/v0.5 cadence: four independently-reviewable PRs, each merged before the
next opens.

## Cross-cutting changes

Workspace impact:

- `Cargo.toml` workspace `members` gains `crates/tool-grep`.
- `[workspace.dependencies]` gains:
  - `tool-grep = { path = "crates/tool-grep", version = "0.6.0" }`
  - `grep-regex = "0.1"` and `grep-searcher = "0.1"` (shared by tool-grep)
  - `ignore = "0.4"` (shared by tool-grep and the rewritten tool-fs glob)
- `crates/savvagent` gains `[[bin]] savvagent-tool-grep` shim.
- `ToolBins` in `crates/savvagent/src/main.rs` gains `grep: Option<PathBuf>`.
- `ToolRegistry::connect` in `crates/savvagent-host/src/tools.rs` sets
  `SAVVAGENT_TOOL_GREP_ROOT` next to the existing two env vars.

Cross-PR rule: each PR rebases off the most recent `master` so review diffs
stay clean. Comments are resolved on the same branch — no chained PRs.

## PR 9 — `tool-grep`

### Crate

`crates/tool-grep`. Library crate mirroring `tool-fs`:

- `crates/tool-grep/Cargo.toml` — depends on `rmcp`, `grep-regex`,
  `grep-searcher`, `ignore`, `serde`, `schemars`, `thiserror`, `tokio`,
  `tracing`, `tracing-subscriber`.
- `crates/tool-grep/src/lib.rs` — `GrepTools`, MCP tool surface, `run()` entry
  point honoring `SAVVAGENT_TOOL_GREP_ROOT`.
- `crates/savvagent/src/bin/savvagent-tool-grep.rs` — 10-line shim around
  `tool_grep::run`, registered as a `[[bin]]` in `crates/savvagent/Cargo.toml`.

### MCP tool surface

One tool, `search`:

```rust
pub struct SearchInput {
    pub pattern: String,                       // regex syntax
    pub path: Option<String>,                  // root, default "."
    pub glob: Option<String>,                  // additional path filter
    pub max_results: Option<u32>,              // default 1024
    #[serde(default)]
    pub case_insensitive: bool,
    #[serde(default)]
    pub multiline: bool,
}

pub struct SearchOutput {
    pub pattern: String,
    pub root: String,
    pub matches: Vec<SearchMatch>,
    pub truncated: bool,
}

pub struct SearchMatch {
    pub file: String,    // path relative to root
    pub line: u32,       // 1-indexed
    pub column: u32,     // 1-indexed, byte column of match start
    pub text: String,    // full matched line, no trailing newline
}
```

`pattern` is compiled once via `grep_regex::RegexMatcherBuilder` at the start
of the call. `glob` is layered onto the walker via `ignore::overrides::OverrideBuilder`.

`max_results` defaults to `DEFAULT_MAX_SEARCH_MATCHES = 1024`, matching the
existing constants on `tool-fs`.

### Containment

Same shape as `FsTools::with_root`:

- `GrepTools::with_root(root)` canonicalizes the project root once.
- Reuses the `path_has_parent_dir` and `is_within` helpers from `tool-fs` —
  these get extracted into a tiny shared module if duplication starts to bite,
  otherwise stay inlined.
- `SAVVAGENT_TOOL_GREP_ROOT` activates containment in the bundled binary.
- **No flag turns containment off.** Layer-1 path containment is non-optional
  for tool-grep per the issue's Inviolable invariants.

### Sensitive-path floor

Server-side filter that drops any match whose path matches:

- `**/.env` and `**/.env.*`
- `**/.ssh/**`
- Case-insensitive substring `credential` in any path component

Filter applies *after* matching, before serializing — even with the deny floor
disabled at the host layer (which won't happen, but belt-and-suspenders), the
matches under those paths never cross the wire.

### Walker

```rust
let walker = ignore::WalkBuilder::new(&resolved_root)
    .git_ignore(true)
    .git_global(true)
    .git_exclude(true)
    .hidden(true)              // skip dotfiles like .DS_Store, .pyc-cache, etc.
    .overrides(globs)          // optional input.glob filter
    .build();
```

Per-file searching uses `grep_searcher::SearcherBuilder::new().multi_line(input.multiline)` against the compiled `RegexMatcher`. Searching is bounded
by `max_results` so the walker can short-circuit once enough matches accumulate.

### Tests

`crates/tool-grep/src/lib.rs` `#[cfg(test)] mod tests`:

- `search_returns_structured_matches` — write three Rust files, search for
  `fn `, assert file/line/column populated.
- `search_case_insensitive` — pattern `HELLO`, file with `hello`, default
  off → empty; with `case_insensitive=true` → match.
- `search_glob_filter` — pattern `fn`, glob `**/*.toml` → no matches even when
  `.rs` files contain `fn`.
- `search_respects_gitignore` — file in gitignored dir excluded by default.
- `search_multiline` — pattern `fn foo\(\)\n` matches across lines only when
  `multiline=true`.
- `search_max_results_truncates` — assert `truncated=true` when limit is
  exceeded.
- `with_root_search_rejects_parent_traversal` — `path: "../"` rejected.
- `with_root_search_filters_symlinked_match` — symlink-to-outside escape
  filtered out.
- `with_root_search_drops_env_files` — populate `.env`, `.env.local`,
  `secrets/credentials.json`, `.ssh/id_rsa`; pattern matches their content;
  assert zero results.

## PR 10 — Structured `tool-edit`

### Tool surface

Three new MCP tools added to `tool-fs`. No new crate, no new env var, no new
spawn.

```rust
// replace
pub struct ReplaceInput {
    pub path: String,
    pub old: String,
    pub new: String,
    pub count: Option<ReplaceCount>,    // None = require unique match
}
pub enum ReplaceCount {
    Exactly(u32),                       // require exactly N matches
    All,                                // replace every occurrence
}
pub struct ReplaceOutput {
    pub path: String,
    pub replacements: u32,
}

// insert
pub struct InsertInput {
    pub path: String,
    pub after_line: u32,                // 0 = prepend; N = after the Nth line
    pub text: String,                   // newline appended if not present
}
pub struct InsertOutput {
    pub path: String,
    pub lines_inserted: u32,
}

// multi_edit
pub struct MultiEditInput {
    pub path: String,
    pub edits: Vec<MultiEdit>,
}
pub enum MultiEdit {
    Replace { old: String, new: String, count: Option<ReplaceCount> },
    Insert  { after_line: u32, text: String },
}
pub struct MultiEditOutput {
    pub path: String,
    pub edits_applied: u32,
}
```

### Replace semantics

- `count: None` → require `old` to appear exactly once. Zero matches errors as
  `not_found`. More than one match errors as `ambiguous` with the line numbers
  of every match in the message so the agent can disambiguate.
- `count: Some(Exactly(n))` → require exactly `n` matches; otherwise error.
- `count: Some(All)` → replace every occurrence; not an error if zero matches
  (the tool returns `replacements: 0`).

### Atomicity (multi_edit)

```text
1. Read file contents → String (via existing read path; respects max_bytes).
2. Apply each edit to the in-memory string in order. If any edit fails
   (e.g. ambiguous replace), abort with no disk writes.
3. Open <path>.tmp.<random> in the same directory with O_CREAT|O_EXCL|O_WRONLY.
4. Write contents, fsync, close.
5. std::fs::rename(<tmp>, <path>) — atomic on POSIX, atomic-enough on Windows.
6. On any failure between (3) and (5), unlink <tmp> and surface the error.
```

This guarantees either every edit lands or none do; the original file is
never observably partial.

### Edge cases

- **Final newline preservation.** Operate on `&str`; rewrites preserve
  trailing newline iff the original had one.
- **Line endings.** Detect on read (`\r\n` if present in the first 64 KiB,
  else `\n`); emit using detected style for `insert`'s injected text.
- **Encoding.** UTF-8 only, same as `read_file` / `write_file`. Non-UTF-8
  files surface `NotUtf8` (existing error variant).

### Implementation

New module `tool-fs/src/edit.rs` containing the three handlers and the shared
atomic-write primitive:

```rust
async fn atomic_write(target: &Path, contents: &[u8]) -> Result<(), FsToolError>
```

Reuses `FsTools::resolve_for_write` for path containment. Sensitive-path floor
(`.env*`, `.ssh/`) is enforced before the in-memory edits run; if the
*resolved* path falls under those, the call errors before any read.

### Tests

In `tool-fs/src/edit.rs`:

- `replace_unique_match` — happy path.
- `replace_zero_matches_errors_not_found`.
- `replace_ambiguous_errors_with_line_numbers`.
- `replace_count_exactly_n` — happy path; off-by-one errors.
- `replace_count_all` — replaces every occurrence; zero is OK.
- `insert_at_zero_prepends`.
- `insert_after_last_line_appends`.
- `insert_beyond_eof_errors`.
- `multi_edit_atomic_on_failure` — file is unchanged after a deliberately bad
  second edit. Inspect parent dir afterward — no leftover `.tmp.*`.
- `multi_edit_round_trip` — three edits applied, file matches expected.
- `tool_edit_rejects_outside_root` — write to `..` rejected (existing
  containment harness).
- `tool_edit_rejects_env_file` — write to `.env` rejected by deny floor.

## PR 11 — Richer `glob` + gitignore

### Walker rewrite

Replace the `glob` crate inside `tool-fs/src/lib.rs` with `ignore::WalkBuilder`:

```rust
let walker = ignore::WalkBuilder::new(&resolved_root)
    .git_ignore(input.respect_gitignore.unwrap_or(true))
    .git_global(input.respect_gitignore.unwrap_or(true))
    .git_exclude(input.respect_gitignore.unwrap_or(true))
    .hidden(false)              // dotfiles can match if pattern says so
    .overrides(
        ignore::overrides::OverrideBuilder::new(&resolved_root)
            .add(&input.pattern)
            .map_err(|e| FsToolError::Glob(e.to_string()))?
            .build()
            .map_err(|e| FsToolError::Glob(e.to_string()))?,
    )
    .build();
```

### Input change

```rust
pub struct GlobInput {
    pub pattern: String,
    pub root: Option<String>,
    pub max_matches: Option<u32>,
    #[serde(default)]
    pub respect_gitignore: Option<bool>,    // None → true
}
```

### Backwards compatibility

- `path_has_parent_dir` rejection on `..` patterns remains unchanged.
- The 5 existing glob tests in `tool-fs/src/lib.rs` are audited:
  - `glob_matches_under_root` — passes unchanged.
  - `with_root_glob_filters_outside_matches` — passes (the `ignore` walker
    doesn't follow symlinks by default; we still keep the canonicalize-and-
    check belt-and-suspenders filter).
  - `with_root_glob_rejects_parent_traversal` — passes unchanged.
  - The 2 other existing `glob_*` tests — audited individually, updated only
    if behavior changes meaningfully.

### New tests

- `glob_excludes_gitignored_by_default` — `.gitignore` listing `target/`,
  pattern `**/*.rs`, file in `target/` excluded.
- `glob_includes_gitignored_when_disabled` — same setup with
  `respect_gitignore=false` includes the file.
- `glob_excludes_dot_git` — `.git/` always excluded (`ignore`'s default).
- `glob_brace_pattern` — `**/*.{rs,toml}` matches both extensions.

### Drop the `glob = "0.3"` dependency from `tool-fs/Cargo.toml`

`ignore` covers all current usage.

## PR 12 — SPP `list_models` + `/model` validation

### Protocol additions

New file `crates/savvagent-protocol/src/models.rs`:

```rust
#[derive(Clone, Debug, Default, Serialize, Deserialize, schemars::JsonSchema)]
pub struct ListModelsRequest {}

#[derive(Clone, Debug, Default, Serialize, Deserialize, schemars::JsonSchema)]
pub struct ListModelsResponse {
    pub models: Vec<ModelInfo>,
}

#[derive(Clone, Debug, Serialize, Deserialize, schemars::JsonSchema)]
pub struct ModelInfo {
    pub id: String,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub display_name: Option<String>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub context_window: Option<u32>,
    #[serde(default)]
    pub default: bool,
}
```

Re-exported from `savvagent-protocol/src/lib.rs`.

### Trait additions

`crates/savvagent-mcp/src/lib.rs`:

```rust
#[async_trait]
pub trait ProviderHandler: Send + Sync {
    async fn complete(&self, ...) -> ...;

    async fn list_models(&self) -> Result<ListModelsResponse, ProviderError> {
        Err(ProviderError {
            kind: ErrorKind::Internal,
            message: "list_models not implemented by this provider".into(),
            retry_after_ms: None,
            provider_code: None,
        })
    }
}

#[async_trait]
pub trait ProviderClient: Send + Sync {
    async fn complete(&self, ...) -> ...;

    async fn list_models(&self) -> Result<ListModelsResponse, ProviderError> {
        Err(ProviderError {
            kind: ErrorKind::Internal,
            message: "provider does not advertise list_models".into(),
            retry_after_ms: None,
            provider_code: None,
        })
    }
}
```

`InProcessProviderClient::list_models` delegates to `handler.list_models()`.

### Provider implementations

- **provider-anthropic**. Hardcoded curated list — Anthropic doesn't expose a
  reliable `/v1/models` endpoint per API key. List is hardcoded in this PR
  covering the current Claude 4.x family (Opus, Sonnet, Haiku) and the `[1m]`
  1M-context variants. The default-flagged entry matches the existing
  `default_model` for the `anthropic` `ProviderSpec`. Test pins the exact list
  — adding a new model is a deliberate one-line change.

- **provider-openai**. `GET /v1/models` against the configured base URL.
  Filter to chat-capable IDs by prefix (`gpt-`, `o1-`, `o3-`, `o4-`). Map to
  `ModelInfo` with `display_name = id`. `default = true` for the configured
  default model.

- **provider-local**. Reuse the `/api/tags` HTTP client already used by the
  readiness probe in `OllamaProvider::ready`. Each tag's `name` becomes a
  `ModelInfo.id`.

- **provider-gemini**. Out of scope for v0.6.0 per the issue. Inherits the
  default trait impl — `/model` falls through to the optimistic path.

### MCP wrappers

Each `{provider}/src/mcp.rs` gets a second `#[tool(name = "list_models")]`:

```rust
#[tool(
    name = "list_models",
    description = "List models this provider can serve."
)]
pub async fn list_models_tool(&self) -> Result<CallToolResult, ErrorData> {
    match self.provider.list_models().await {
        Ok(resp) => {
            let value = serde_json::to_value(&resp).map_err(...)?;
            Ok(CallToolResult::structured(value))
        }
        Err(spp_err) => {
            let value = serde_json::to_value(&spp_err).map_err(...)?;
            Ok(CallToolResult::error_structured(value))
        }
    }
}
```

### Streamable HTTP `ProviderClient`

The host-side `RmcpProviderClient` (in `crates/savvagent-host/src/provider.rs`)
gains a `list_models` impl that issues an MCP `tools/call` for `list_models`.
If the provider's `tools/list` doesn't advertise it, the call returns the
default trait error (treated as "fall through to optimistic" by the host).

### Host plumbing

`crates/savvagent-host/src/lib.rs` `Host`:

```rust
pub async fn list_models(&self) -> Result<ListModelsResponse, ProviderError> {
    self.provider.list_models().await
}
```

### TUI `/model` change

`crates/savvagent/src/main.rs` `handle_model_command` validates against
`host.list_models()` before swapping:

```rust
// rest is non-empty here
let host = match current_host(host_slot).await {
    Some(h) => h,
    None => { app.push_note("Not connected — `/connect` first."); return; }
};
match host.list_models().await {
    Ok(resp) => {
        if !resp.models.iter().any(|m| m.id == rest) {
            let known: Vec<_> = resp.models.iter().map(|m| m.id.as_str()).collect();
            app.push_note(format!(
                "Unknown model `{rest}`. Known: {}",
                known.join(", ")
            ));
            return;
        }
    }
    Err(e) => {
        // Optimistic fallback — provider doesn't advertise list_models.
        tracing::debug!(?e, "list_models unsupported, proceeding optimistically");
    }
}
perform_model_change(...).await;
```

### Tests

- Unit tests on each provider's `list_models` impl (mock HTTP for openai, real

- Integration test: spin up the in-process provider, call `host.list_models()`,
  assert the curated list comes back.
- TUI command-routing test (extends the existing test harness): unknown model

## Inviolable invariants

- `tool-grep` honors the `.env*` / `.ssh/` deny floor server-side, before
  matches are serialized.
- `tool-grep` Layer-1 path containment is non-optional. No flag, no env var,
  no input field disables it.
- `tool-edit` writes are atomic — either every edit in a batch lands or none.
- `tool-edit` honors the same deny floor as `tool-fs`.

## Review pipeline (per PR)

Before merging each PR, dispatch in parallel:

- `pr-review-toolkit:code-reviewer`
- `pr-review-toolkit:silent-failure-hunter`
- `pr-review-toolkit:pr-test-analyzer`
- `pr-review-toolkit:type-design-analyzer` (PR 12 especially — trait change)
- `pr-review-toolkit:comment-analyzer`
- `pr-review-toolkit:code-simplifier`

Resolve every actionable finding on the same branch, then merge.
