# Localize TUI — v0.10.0 design

Date: 2026-05-13
Status: pending review
Roadmap: v0.10.0 (#31)
Supersedes: nothing
Related: v0.9 plugin system (#36, #37, #38, #39, #41, #43, #44, #45)

## Problem

Every user-facing string in the TUI is a hardcoded English literal. Across
`crates/savvagent/src/` this includes:

- The startup splash (tagline, sandbox advisory, hint, version line) —
  now owned by the `internal:splash` plugin.
- The themes picker chrome (modal title `"Pick a theme"`, the
  `format!("no themes match \`{}\`", ...)` body line, the `tips()` line
  `"↑/↓ navigate · type to filter · Enter apply · Esc cancel"`, the
  `format_listing` slash output, the section labels `"  built-in:"` and
  `"  catalog (ratatui-themes):"`, and the `theme set to \`...\``
  persistence note).
- Equivalent chrome inside every other picker plugin: `internal:connect`,
  `internal:resume`, `internal:view-file`, `internal:edit-file`,
  `internal:plugins-manager`, `internal:command-palette`.
- Slot content from `internal:home-footer` and `internal:home-tips`.
- Slash-command `summary` fields populated at plugin manifest construction
  time (rendered by `internal:command-palette` as the second column).
- All `app.push_note(...)` / `app.push_styled_note(...)` text emitted
  from the effect applier and from the in-tree `dispatch_slash_command`
  path that still handles `/bash`, `/sandbox`, `/tools`.
- Modal titles set in `ScreenLayout::CenteredModal { title: Some("...") }`
  for each plugin's `ScreenSpec`.
- The "you/agent" role prefixes in the conversation log render path.

There is no i18n layer today. Issue #31 specifies the framework
(`rust-i18n`), four shipped languages (English plus Spanish, Portuguese,
Hindi), and a `/language` slash picker modeled on the existing theme
picker. Per `feedback_ui_via_plugins`, the picker must be a plugin.

## Approach

The themes picker (`crates/savvagent/src/plugin/builtin/themes/`, shipped
in v0.9 PR 6) is the canonical picker-as-a-plugin pattern in this
codebase. The language picker mirrors it directly: same module shape
(`mod.rs` + `picker.rs` + `screen.rs` + `catalog.rs`), same outcome enum
(`Stay`/`Preview`/`Apply`/`Cancel`), same screen lifecycle, same
`open_screen` placeholder-then-patch trick for reading App state.

Three pieces ship in v0.10.0:

1. **Catalog + i18n macro inside `savvagent`.** A new
   `crates/savvagent/locales/` directory with one TOML per shipped
   locale (`en.toml`, `es.toml`, `pt.toml`, `hi.toml`). The savvagent
   crate declares `rust_i18n::i18n!("locales", fallback = "en")` at the
   top of `main.rs`; every user-facing string in the crate is rewritten
   to `t!("key")` or `t!("key", var = value)`. `rust-i18n = "3.1"` is
   already a workspace dependency — only the per-crate
   `rust-i18n.workspace = true` line is new.
2. **`crates/savvagent/src/plugin/builtin/language/catalog.rs`** —
   the language catalog + persistence helpers, mirroring
   `themes::catalog`: env-var detection, supported languages,
   `~/.savvagent/language.toml` load/save.
3. **`internal:language` built-in plugin** at
   `crates/savvagent/src/plugin/builtin/language/` mirroring the
   themes plugin layout. The plugin contributes the `/language` slash
   command and one `Screen` (centered modal), and emits new
   `Effect::SetActiveLocale { code, persist }` instances.

All global state mutation (the `rust_i18n` locale, the TOML file) lives
in the runtime's effect applier (`crates/savvagent/src/plugin/effects.rs`).
The plugin returns effects; it does not touch globals or the
filesystem. This mirrors the themes plugin / `Effect::SetActiveTheme`
contract exactly.

### Two additive variants on existing `#[non_exhaustive]` enums

The plugin-crate (`savvagent-plugin`) changes for this work are two
additive enum variants — no breaking changes:

- `Effect::SetActiveLocale { code: String, persist: bool }` on
  `enum Effect` (already `#[non_exhaustive]`).
- `ScreenArgs::LanguagePicker { current_code: String }` on
  `enum ScreenArgs` (already `#[non_exhaustive]`), with a new arm in
  the existing `screen_id()` mapping returning `Some("language.picker")`.

No `ScreenSpec` or `SlashSpec` field renames. The in-flight-turn gate
that an earlier draft of this spec proposed (`blocks_during_turn` on
`ScreenSpec`) is dropped: master's themes picker does not gate, and the
language picker is consistent with it. The `/bash` and `/resume`
in-flight gates that exist today are tactical checks inside
`dispatch_slash_command` because they mutate non-UI state; the language
picker only swaps a runtime constant and a file, so the same
no-gate rule applies.

### Slash-summary localization — known limitation

`SlashSpec::summary` is populated when a plugin's `manifest()` is called
once at registration time (in `register_builtins()`). The command
palette renders that snapshot verbatim. If we call `t!("...")` inside
`manifest()`, the string captures whatever locale was active at
registration — which is the user's saved locale after boot, because
`rust_i18n::set_locale(&detect_initial())` runs before
`register_builtins()`.

Consequence: changing language mid-session via `/language es` does not
update slash-command summaries in the command palette until the next
process restart. The summary column shows the previously-selected
language's wording for one session.

This is an acceptable trade-off for v0.10.0: the alternative is
renaming `SlashSpec::summary` (breaking field rename on a v0.9 type
used by 14 in-tree plugins, plus a re-resolve path through manifests
indexes) for a mostly-cosmetic concern. Future work can plumb dynamic
`t!()` at palette render time if it bites.

All other strings (modal titles, picker rows, notes pushed via Effects,
status-bar text) re-resolve every frame because they call `t!()` inside
the render path or just before pushing.

### Why not migrate the remaining in-tree slash commands?

Out of scope. v0.9 already moved the theme, connect, resume, view-file,
edit-file, plugins-manager, and command-palette pickers into plugins.
The slash commands that still live in `dispatch_slash_command`
(`/bash`, `/sandbox`, `/tools`, and a few others depending on current
state) are not part of this work. Localizing their *strings* via
`t!()` is in scope (they live in `crates/savvagent/src/main.rs`);
changing where those commands live is not.

## Components

### `crates/savvagent/locales/{en,es,pt,hi}.toml`

One file per locale, sections by surface, dotted keys flat under
`t!()`. Illustrative excerpt — the full key inventory is enumerated in
the implementation plan, not in this design:

```toml
# locales/en.toml
[notes]
theme-set            = "theme set to `%{slug}`"
theme-not-found      = "theme `%{slug}` not found — run `/theme list` to see available themes."
language-set         = "language set to %{native}"
language-not-found   = "language `%{code}` not supported — run `/language` to pick one."
persistence-failed   = "applied for this session, but persistence failed: %{err}"
cannot-during-turn   = "Cannot %{cmd} during an in-flight turn — wait for it to finish."

[splash]
tagline              = "your terminal coding agent"
hint                 = "press / for commands"

[picker.themes]
modal-title          = "Pick a theme"
filter-label         = "Filter:"
no-match             = "no themes match `%{filter}`"
section-builtin      = "  built-in:"
section-catalog      = "  catalog (ratatui-themes):"
tips                 = "↑/↓ navigate · type to filter · Enter apply · Esc cancel"
listing              = "Themes (%{count}): %{names}"

[picker.language]
modal-title          = "Pick a language"
filter-label         = "Filter:"
no-match             = "no languages match `%{filter}`"
tips                 = "↑/↓ navigate · type to filter · Enter apply · Esc cancel"
listing              = "Languages (%{count}): %{names}"

[picker.connect]
modal-title          = "Connect provider"
# ... per-picker chrome (one section per picker plugin)

[footer]
turn-working         = "turn #%{id} working"
# ... home-footer slot content

[tips]
press-slash          = "Press / for commands"

[slash]

# ... one summary key per registered slash command
```

Naming convention: `<surface>.<descriptor>` where surfaces are
`notes`, `splash`, `picker.<plugin-id-suffix>`, `footer`, `tips`,
`slash`, `errors`. Descriptors are kebab-case action or state names,
never English phrases. Interpolation variables (`%{slug}`, `%{code}`,
`%{err}`, `%{id}`) match parameter names at call sites.

`handle_slash` (mirrors `ThemesPlugin::handle_slash`):

| Args                | Returned effects                                                                                  |
|---------------------|---------------------------------------------------------------------------------------------------|
| `[]`                | `[OpenScreen { id: "language.picker", args: ScreenArgs::LanguagePicker { current_code: <placeholder> } }]` |
| `["list"]`          | `[PushNote(StyledLine::plain(t!("picker.language.listing", count = .., names = ..)))]`            |
| `["es"]`            | `[SetActiveLocale { code: "es", persist: true }]`                                                 |
| `["xx"]` (unknown)  | `[PushNote(StyledLine::plain(t!("notes.language-not-found", code = "xx")))]`                      |

`current_code` is a placeholder (e.g. `"en"`); `apply_effects::open_screen`
patches it to `app.active_language` before constructing the screen —
exactly the pattern the themes plugin uses for `current_slug`.

`create_screen`:

```rust
fn create_screen(&self, id: &str, args: ScreenArgs) -> Result<Box<dyn Screen>, PluginError> {
    match (id, args) {
        ("language.picker", ScreenArgs::LanguagePicker { current_code }) => {
            let current = catalog::lookup(&current_code).copied()
                .unwrap_or_else(|| {
                    tracing::warn!(code = %current_code,
                        "language.picker received unknown code; defaulting to English");
                    catalog::lookup("en").copied().expect("en always supported")
                });
            Ok(Box::new(LanguagePickerScreen::new(LanguagePicker::new(current))))
        }
        (other, _) => Err(PluginError::ScreenNotFound(other.to_string())),
    }
}
```

`LanguagePicker` (in `picker.rs`) mirrors `ThemePicker`:

```rust
pub(crate) struct LanguagePicker {

    pub(crate) cursor: usize,
    pub(crate) pre_open_code: &'static str,

pub(crate) enum PickerOutcome {

    PreviewLocale(&'static str),       // code only; render-time lookup
```

### `crates/savvagent/src/app.rs` (modified)

Add one field and two methods, mirroring `active_theme` /
`set_active_theme_by_slug` / `persist_config`:

```rust
/// Currently-active locale code (e.g. `"en"`). Loaded at startup
/// from `~/.savvagent/language.toml` (or env detection); mutated by
/// `apply_effects` on `Effect::SetActiveLocale`.
pub active_language: String,
```

```rust
/// Set the active locale by code. Unknown codes are surfaced as a
/// styled note; the in-memory selection (and the `rust_i18n` global)
/// are left unchanged. Called from `apply_effects` on `Effect::SetActiveLocale`.
pub fn set_active_language(&mut self, code: String) {
    if crate::plugin::builtin::language::catalog::is_supported(&code) {
        rust_i18n::set_locale(&code);
        self.active_language = code;
    } else {
        self.push_styled_note(savvagent_plugin::StyledLine::plain(
            rust_i18n::t!("notes.language-not-found", code = code).to_string()
        ));
    }
}

/// Persist the active locale to `~/.savvagent/language.toml`. Errors
/// surface as a styled note; the in-memory selection is kept either
/// way. Called from `apply_effects` on `Effect::SetActiveLocale { persist: true }`.
pub fn persist_language(&mut self) {
    let code = self.active_language.clone();
    match crate::plugin::builtin::language::catalog::save(&code) {
        Ok(()) => {
            let native = crate::plugin::builtin::language::catalog::lookup(&code)
                .map(|l| l.native_name).unwrap_or(code.as_str());
            self.push_styled_note(savvagent_plugin::StyledLine::plain(
                rust_i18n::t!("notes.language-set", native = native).to_string()
            ));
        }
        Err(e) => {
            self.push_styled_note(savvagent_plugin::StyledLine::plain(
                rust_i18n::t!("notes.persistence-failed", err = format!("{e}")).to_string()
            ));
        }
    }
}
```

`App::new` accepts `active_language: String` and stores it. Existing
`App::new` callers (tests, real construction in `main.rs`) pass the
boot-time value.

## Flow

### Boot

```text
main()
  ├── crate::plugin::builtin::language::catalog::detect_initial()
  │     ├── load() → ~/.savvagent/language.toml (file wins if value is supported)
  │     ├── normalize_env_locale($LC_ALL, $LC_MESSAGES, $LANG)
  │     └── "en"
  ├── rust_i18n::set_locale(&initial_code)
  ├── App::new(..., active_language: initial_code, ...)
  └── register_builtins()
        └── each plugin's manifest() runs with locale already set
            (so SlashSpec.summary and ScreenLayout.title capture the
             initial locale; mid-session /language won't refresh them)
```

### `/language` (no args)

```text
user types "/language", Enter
  → SlashRouter dispatches to LanguagePlugin::handle_slash
  → returns vec![OpenScreen { id: "language.picker",
                              args: LanguagePicker { current_code: "en" /* placeholder */ } }]
  → apply_effects::open_screen patches args:
       LanguagePicker { current_code: app.active_language.clone() }
  → registry.get(internal:language).create_screen → LanguagePickerScreen
  → screen_stack.push(...)
  → next frame: ratatui draws the centered modal.
    Modal title comes from t!("picker.language.modal-title").
    Rows come from catalog::supported().
    Tips line from t!("picker.language.tips").
```

### `/language es`

```text
LanguagePlugin::handle_slash("language", ["es"])
  → returns vec![SetActiveLocale { code: "es", persist: true }]
  → apply_effects:
      app.set_active_language("es")
         // rust_i18n::set_locale("es"); app.active_language = "es"
      app.persist_language()
         // catalog::save("es");
         // push_styled_note(t!("notes.language-set", native = "Español"))
  → next frame: all chrome re-renders in Spanish (except slash summaries —
    see Slash-summary localization limitation above)
```

### In-modal navigation + Enter

```text
↓
  → Screen::on_key → picker.on_key → PickerOutcome::PreviewLocale("pt")
  → returns vec![SetActiveLocale { code: "pt", persist: false }]
  → apply_effects: app.set_active_language("pt"); no persist call
  → next frame: chrome behind modal is Portuguese; modal title also Portuguese

Enter
  → PickerOutcome::Apply("pt")
  → returns vec![Stack(SetActiveLocale { code: "pt", persist: true }, CloseScreen)]
  → apply_effects iterates Stack:
      → SetActiveLocale: set_active_language + persist_language
      → CloseScreen: screen_stack.pop()
```

### Esc

```text
Esc
  → PickerOutcome::Cancel
  → returns vec![Stack(SetActiveLocale { code: pre_open_code, persist: false }, CloseScreen)]
  → apply_effects: locale reverts in memory, no file write; modal closes
```

## Slash-command surface

| Invocation                            | Behavior                                                                                          |
|---------------------------------------|---------------------------------------------------------------------------------------------------|
| `/language`                           | `OpenScreen { id: "language.picker", .. }`. No inline output.                                     |
| `/language list`                      | `PushNote` with `t!("picker.language.listing", count = .., names = ..)`.                          |
| `/language <code>`                    | `SetActiveLocale { code, persist: true }`. Note: `t!("notes.language-set", native = ..)`.         |
| `/language <unknown>`                 | `PushNote(t!("notes.language-not-found", code = <input>))`. Active locale unchanged.              |
| Command palette → `Execute("/language")` | Same as `/language` no-args; opens the picker.                                                |

In-flight-turn handling: no gate. The themes picker is the precedent —
it does not refuse during turns. `/bash` and `/resume` retain their
tactical pre-flight refusals in `dispatch_slash_command` because they
mutate non-UI state; the language picker only swaps a runtime constant
and a file.

## String migration — inventory pass

During plan-writing, the implementer enumerates every user-facing
string in `crates/savvagent/src/` via the grep seeds below and assigns
each a key. The plan is the authoritative inventory; this design fixes
the conventions.

Grep seeds:

```text
push_note\(                  # app.rs, main.rs
push_styled_note\(           # app.rs, plugin builtins, effects.rs
StyledLine::plain\(          # plugin builtins (every picker chrome)
.title\(                     # any remaining ui.rs / main.rs modal/border titles
title: Some\("              # ScreenSpec literal titles in plugin manifests
description: "              # Manifest descriptions
summary: "                  # SlashSpec summaries (all plugins)
Span::styled\(               # ui.rs literal spans
Line::raw\(                  # ui.rs literal lines
Paragraph::new\(             # ui.rs body text
plugin/builtin/.*/mod.rs     # every built-in plugin
plugin/builtin/.*/screen.rs  # every screen impl
```

Per-plugin sweep targets (one task per plugin/file in the plan):

- `internal:home-footer` slot output (sandbox label, turn state, working dir, key reminder).
- `internal:home-tips` slot output (`"Press / for commands"`, `"Connecting…  Press / for commands"`).
- `internal:splash` plugin content (tagline, hint, sandbox advisory lines).
- `internal:command-palette` empty-state and any inline labels.
- `internal:themes` (modal title, picker section headers, `format_listing`, `tips()`, `theme set to` / `theme not found` notes inside `app.set_active_theme_by_slug` and `persist_config`).
- `internal:connect` modal title + body labels.
- `internal:resume` modal title + body labels + "no transcripts found" empty state.
- `internal:view-file` / `internal:edit-file` modal titles + status lines.
- `internal:plugins-manager` modal title + column labels + toggle hint.
- `internal:model` slash output (notes only).
- `internal:save`, `internal:clear`, `internal:quit` slash output.
- `dispatch_slash_command` and helpers in `main.rs`: `/bash`, `/sandbox`, `/tools` notes and prompts.
- `ui.rs` conversation-log role prefixes (`"You: "`, `"Agent: "`) and any other literal strings.

Out of scope (stays English in this PR):

- `eprintln!` to stderr (diagnostic; not part of the rendered TUI).
- Tool crates (`tool-fs`, `tool-bash`, `tool-grep`) and their schemas.
- Provider crates (`provider-anthropic`, `provider-gemini`, `provider-openai`, `provider-local`).
- `anyhow` error chains from upstream crates (envelope localizes; chain stays raw).
- Panic messages, `unwrap` failures, Rust runtime output.

Conversion rules:

1. **Bare literal:** `"Pick a theme"` → `t!("picker.themes.modal-title")`.
2. **`format!` with named or positional args:** convert to `t!()` with named interpolation. `format!("theme set to `{slug}`")` → `t!("notes.theme-set", slug = slug)`.
3. **Strings wrapping user-supplied content:** envelope localizes, content stays raw. `t!("notes.save-error", err = format!("{e:#}"))`.
4. **Per-line `Vec<StyledLine>` builds:** one key per visual line. Don't try to localize sub-phrases inside a single line.
5. **Manifest fields (`summary`, `description`, `name`, `ScreenLayout` titles):** call `t!()` inside `manifest()`. Captures the boot-time locale; restart to refresh after mid-session `/language` switch. Documented limitation.

## Edge cases

1. **`language.toml` missing.** `load()` returns `None`; fall through to env detection.
2. **`language.toml` parse error.** `load()` returns `None`; one-line `eprintln!` warning; fall through to env detection.
3. **`language.toml` contains unsupported code.** Same as parse error path: warn, fall through to env, eventually to `en`.
4. **`$LANG = "C"` or `"POSIX"`.** `normalize_env_locale` returns `None`; skip.
5. **`$LANG = ""`.** Returns `None`; skip.
6. **`$LANG = "ru_RU.UTF-8"` (unsupported).** Normalize returns `Some("ru")`; `is_supported` returns false; fall through to next var, then `en`.
7. **`$HOME` unset.** `config_path()` returns `None`; `load` returns `None`; `save` is a silent no-op (mirrors `themes::catalog`). Session works; choice doesn't persist.
8. **User picks `hi` and terminal font lacks Devanagari.** Visual degradation only — rows render with replacement glyphs. The `code` column (`hi`) is always ASCII, so the user can still filter. Documented as a font requirement.
9. **Live-preview swaps the modal's own title during /language picker.** Intentional and ergonomic — the user sees the language they're considering. The title re-resolves every frame because `Screen::render` calls `t!()`.
10. **User runs `/language` then Esc immediately.** Pre-code equals active code; the reverting `SetActiveLocale` is a no-op write to `rust_i18n` and no note is pushed.
11. **Filter narrows to one language, Enter.** Commits that one.
12. **Filter narrows to zero, Enter.** No-op (`PickerOutcome::Stay`). Esc still cancels.
13. **Filter shrinks below current index.** Picker's `clamp_after_filter_change` brings `cursor` to `filtered.len() - 1` and re-emits `PreviewLocale`.
14. **Slash summaries stale after mid-session `/language`.** Documented limitation; restart refreshes.
15. **Some other plugin emits `SetActiveLocale { code: "xx" }` with an unsupported code.** `app.set_active_language` validates against `catalog::is_supported`; on miss, pushes `notes.language-not-found` and does not call `rust_i18n::set_locale`.
16. **Missing key at runtime.** rust-i18n returns the key string verbatim. The CI parity test (below) prevents this for keys present in `en` but missing in another locale; the only remaining path is a typo at a call site that uses a key no locale has, which `cargo test` catches because `t!()` calls are exercised by the picker plugin tests + the locale-parity test.

## Testing

### Per-locale catalog parity (CI gate)

`crates/savvagent/tests/locales.rs`:

- Loads each shipped TOML via `include_str!` of `en`, `es`, `pt`, `hi`; flattens to a sorted `Vec<String>` of dotted keys.
- Asserts every non-`en` locale has the same key set as `en`; failing pair is reported with the symmetric difference.
- For each key, parses `%{var}` placeholders from the `en` value and asserts the same placeholder *set* appears in the corresponding value in every other locale.

### `normalize_env_locale` unit tests (inside `language/catalog.rs`)

Table-driven:

| Input              | Output            |
|--------------------|-------------------|
| `"es_ES.UTF-8"`    | `Some("es")`      |
| `"pt_BR@euro"`     | `Some("pt")`      |
| `"en-US"`          | `Some("en")`      |
| `"C"` / `"POSIX"`  | `None`            |
| `""`               | `None`            |
| `"hi"`             | `Some("hi")`      |

### Catalog round-trip + detect-initial tests

Mirrors `themes::catalog` tests, under the existing `HOME_LOCK` / `HomeGuard` pattern shared via `crate::test_helpers`:

- `save("es")` + `load()` round-trips.
- Missing file → `None`.
- Malformed TOML → `None`.
- Unknown code in file → `None` + warning logged.
- `$HOME` unset → `save` is `Ok(())` no-op; `load` is `None`.
- File `language="es"`, `LANG=fr_FR.UTF-8` → `detect_initial` returns `"es"`.
- No file, `LC_ALL=pt_BR.UTF-8` → `"pt"`.
- No file, `LC_ALL=ru_RU`, `LANG=hi_IN.UTF-8` → `"hi"`.
- No file, no env → `"en"`.

### `LanguagePlugin::handle_slash` tests

Mirrors `themes::ThemesPlugin` test patterns:

- `handle_slash("language", [])` → `[OpenScreen { id: "language.picker", .. }]`.
- `handle_slash("language", ["list"])` → `[PushNote(..)]`.
- `handle_slash("language", ["es"])` → `[SetActiveLocale { code: "es", persist: true }]`.
- `handle_slash("language", ["xx"])` → `[PushNote(..)]`.
- `manifest()` declares one slash and one screen with the expected ids.

### `LanguagePickerScreen::on_key` tests

Mirrors `themes::screen` test patterns:

- `Enter` → `Stack(SetActiveLocale{persist:true}, CloseScreen)`.
- `Esc` → `Stack(SetActiveLocale{code:pre_open, persist:false}, CloseScreen)`.
- `↓` → `SetActiveLocale{persist:false}` previewing the next code.
- Typed chars filter; `clamp_after_filter_change` re-emits or `Stay`s.

### `apply_effects` tests

In `crates/savvagent/src/plugin/effects.rs` tests, adjacent to existing theme-applier tests:

- `SetActiveLocale { code: "es", persist: true }` → `rust_i18n::locale() == "es"`, `~/.savvagent/language.toml` contains `language = "es"`, success note pushed.
- `SetActiveLocale { code: "es", persist: false }` → locale switched in memory; no file write; no note.
- `SetActiveLocale { code: "xx", persist: true }` → no-op for both `rust_i18n` and the file; `notes.language-not-found` note pushed.
- `open_screen` patcher rewrites `ScreenArgs::LanguagePicker.current_code` to `app.active_language`.

### Manifest registration smoke

Extend `crates/savvagent/src/plugin/mod.rs::register_builtins_pr8_complete`:

- Expected non-provider plugin ids list adds `"internal:language"`.
- `set.plugins.len()` assertion bumps from 14 to 15.
- `reg.len()` assertion bumps accordingly.

### What this design deliberately does NOT test

- Translation *quality* — human review concern. CI verifies structural parity.
- Terminal font rendering of Devanagari — outside the harness.
- `LANGUAGE=` GNU/glibc compound-locale env var — not honored; documented as POSIX-only (`LC_ALL`, `LC_MESSAGES`, `LANG`).
